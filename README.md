# Neighbus Application
**지역 커뮤니티 플랫폼의 React Native 모바일 앱**  
**담당**: iOS/Android 크로스플랫폼 앱 개발

---

## 내가 한 일

**React Native로 iOS와 Android 앱을 동시에 개발했습니다.**

- **React Native + TypeScript**: 단일 코드베이스로 iOS/Android 동시 개발
- **UI/UX 설계**: 웹의 기능을 모바일에 최적화하여 재구현
- **API 연동**: 웹 백엔드 API를 앱에서 사용 가능하도록 통합
- **상태 관리**: Context API와 AsyncStorage로 효율적 상태 관리

**결과**: 웹과 동일한 기능을 iOS/Android 네이티브 앱으로 제공

---

## 주요 기능

| 기능 | 설명 |
|------|------|
| **로그인/회원가입** | AsyncStorage 기반 자동 로그인 |
| **홈** | 카테고리별 동아리 목록, 신규 모임, 인기 게시글 |
| **지도** | Google Maps + Geocoding으로 모임 장소 지정 |
| **동아리** | 동아리 검색, 가입, 상세 정보 |
| **모임** | 오프라인 모임 생성 및 참여 |
| **실시간 채팅** | WebSocket 기반 실시간 메시지 |
| **게시판** | 자유게시판, 갤러리, 공지사항 |
| **마이페이지** | 내 정보 수정, 가입 동아리 관리 |

---

## 🛠 기술 스택

**Frontend**
- React Native + TypeScript
- React Navigation (Bottom Tab + Stack)
- Context API (전역 상태 관리)
- AsyncStorage (로컬 저장소)

**Backend 연동**
- REST API 호출 (fetch)
- WebSocket (실시간 통신)
- Google Maps API (지도 & 지오코딩)

**Platform Support**
- iOS (Swift 네이티브 모듈)
- Android (Java 네이티브 모듈)

**Development**
- TypeScript (타입 안정성)
- ESLint + Prettier (코드 품질)

## 배운 점

### 크로스플랫폼 개발의 복잡성

**상황**: 웹 코드를 그대로 모바일에 적용할 수 없음  
**해결**: 플랫폼별 차이를 이해하고 최적화된 구현  
**결과**: iOS/Android 모두에서 동일한 UX 제공

### 모바일 환경의 차이

**상황**: 웹과 앱의 화면 크기, 입력 방식, 네비게이션이 다름  
**해결**: 모바일 첫 사고방식(Mobile-First)으로 UI/UX 설계  
**결과**: 직관적이고 효율적인 모바일 앱 완성

### 네이티브 모듈 관리

**상황**: iOS/Android 플랫폼별 의존성 관리 필요  
**해결**: 팀원과의 협력으로 플랫폼별 최적화  
**결과**: 안정적이고 성능 우수한 앱 개발
